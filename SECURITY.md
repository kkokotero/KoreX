# Security Policy

## Supported versions

Security fixes are provided for the latest stable release and the current development branch.

## Reporting a vulnerability

If you believe you have found a security issue in KoreX, do **not** open a public issue.

Instead, use the repository's private vulnerability reporting flow or the private contact path available to the maintainers.

Include:

- a clear description of the issue
- affected versions or branches
- steps to reproduce
- relevant logs, stack traces, or proof of concept code
- any suggested mitigation, if available

## What to expect

We will:

1. acknowledge the report
2. investigate the issue
3. coordinate a fix or mitigation
4. follow up with disclosure guidance when appropriate

## Scope

This policy applies to:

- the KoreX library
- published artifacts
- SampleApp when it affects the public release
- build and publishing configuration
